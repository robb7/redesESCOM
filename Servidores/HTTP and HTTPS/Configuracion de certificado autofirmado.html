Configuracion de certificado autofirmado

Adm Servicios en Red


Este tutorial tiene como objetivo la creacion de un certificado autofirmado medianete openssl
Para crear la llave autofirmada y el certificado con OpenSSL utilizamos el siguiente comando: 

sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/ssl/private/apache-selfsigned.key -out /etc/ssl/certs/apache-selfsigned.crt

Una vez

Se le har una serie de preguntas. Antes de repasar eso, echemos un vistazo a lo que est sucediendo en el comando que estamos emitiendo:

openssl : esta es la herramienta bsica de lnea de comandos para crear y administrar certificados, claves y otros archivos de OpenSSL.

req :este subcomando especifica que queremos utilizar la gestin de la solicitud de firma de certificado (CSR) X.509. El X.509 es un estndar de infraestructura de clave pblica al que se adhieren SSL y TLS para su gestin de claves y certificados. Queremos crear un nuevo certificado X.509, por lo que estamos usando este subcomando.

x509 : Esto modifica an ms el subcomando anterior al indicar a la utilidad que queremos hacer un certificado autofirmado en lugar de generar una solicitud de firma de certificado, como ocurre normalmente.

nodos : Esto le dice a OpenSSL que omita la opcin para asegurar nuestro certificado con una frase de contrasea. Necesitamos que Apache pueda leer el archivo, sin la intervencin del usuario, cuando se inicia el servidor. Una frase de contrasea evitara que esto sucediera porque tendramos que ingresarla despus de cada reinicio.

days 365 : esta opcin establece el perodo de tiempo durante el cual el certificado se considerar vlido. Aqu lo ponemos por un ao.

newkey rsa: 2048 : Esto especifica que queremos generar un nuevo certificado y una nueva clave al mismo tiempo. No creamos la clave necesaria para firmar el certificado en un paso anterior, por lo que debemos crearlo junto con el certificado. La porcin rsa:2048 le dice que haga una clave RSA que tenga 2048 bits de longitud.

keyout: esta lnea le dice a OpenSSL dnde colocar el archivo de clave privada generado que estamos creando.

out : Esto le dice a OpenSSL dnde colocar el certificado que estamos creando.

Configuracion del certificado en el servidor de Apache

Archivo de Configuracion:
Ruta: /etc/apache2/sites-enabled/desfault-ssl.conf 

SSLEngine on 

SSLCertificateFile  /etc/ssl/certs/ssl-cert-snakeoil.pem

SSLCertificateKeyFile /etc/ssl/private/ssl-cert-snakeoil.key

Debemos agregar la siguiente lineas al archivo para utilizar la configuracion de SSL

Habilitar los cambios en Apche

Ahora que hemos realizado nuestros cambios, podemos habilitar los mdulos de encabezados y SSL en Apache, habilitar nuestro Host virtual listo para SSL y reiniciar Apache.

Podemos habilitar mod_ssl , el mdulo SSL de Apache y mod_headers , necesarios para algunas de las configuraciones en nuestro fragmento de a2enmod SSL, con el comando a2enmod :

  sudo a2enmod ssl

  sudo a2enmod encabezados

A continuacin, podemos habilitar nuestro Host Virtual SSL con el comando a2ensite :

 sudo a2ensite default-ssl

Tambin necesitaremos habilitar nuestro archivo ssl-params.conf para leer los valores que configuramos:

 sudo a2enconf ssl-params

En este punto, nuestro sitio y los mdulos necesarios estn habilitados. Debemos verificar que no haya errores de sintaxis en nuestros archivos. Podemos hacer esto escribiendo:

  sudo apache2ctl configtest

Si todo tiene xito, obtendrs un resultado que se ver as:

  Salida 

  AH00558: apache2: No se pudo determinar de manera confiable el nombre de dominio completo del servidor, usando 127.0.1.1.  Establezca la directiva 'ServerName' globalmente para suprimir este mensaje

 Sintaxis ok

La primera lnea es solo un mensaje que le indica que la directiva ServerName no est configurada globalmente. Si desea deshacerse de ese mensaje, puede configurar ServerName para el nombre de dominio o la direccin IP de su servidor en /etc/apache2/apache2.conf . Esto es opcional ya que el mensaje no har dao.

Si su salida tiene Syntax OK , su archivo de configuracin no tiene errores de sintaxis. Podemos reiniciar con seguridad Apache para implementar nuestros cambios:

Reiniciar apache

sudo systemctl restart apache2

Redireccionamiento al puerto https
