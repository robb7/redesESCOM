Configuración de SSH

Instalación de SSH en Linux

Introducción

SSH(secure shell) es un protocolo de red para la transferencia de información de manera segura, servicios shell remotos, la ejecución de comandos y otras clases de conexión El servidor ssh corre en una maquina mientras que el cliente se encuentra en otra.

Comando de instalación
$sudo apt-get install ssh

Es importante recordar que para poder realizar la conexión de manera correcta es necesario contar con shh instalado tanto en el cliente como en el servidor.

Conexion a por terminal (Linux a Linux)

La forma basica de conexion una vez teniendo instalado el cliente y el servidor es: 

$ssh "host remoto"

El host remoto en este ejemplo es la dirección IP o el nombre de dominio al que está intentando conectarse .
Este comando supone que su nombre de usuario en el sistema remoto es el mismo que su nombre de usuario en su sistema local.
Si su nombre de usuario es diferente en el sistema remoto, puede especificarlo utilizando esta sintaxis: 

$ssh  "username remoto"@"host remoto"
Imagen de conexión

Bajo el siguiente comando es posible conectarse mediante ssh con la posibilidad de iniciar aplicaciones en modo gráfico 

ssh  -X  ip

Intercambio de llaves para la conexión

Las claves SSH proporcionan una forma fácil y segura de iniciar sesión en su servidor y se recomiendan para todos los usuarios.

El primer paso es la generacion de las llaves SSH por parte del cliente 

ssh-keygen

Después de ingresar el comando, debería ver el siguiente resultado:

Enter file in which to save the key (/home/robb/.ssh/id_rsa): 

Enter passphrase (empty for no passphrase): 

Enter same passphrase again: 

Your identification has been saved in /home/robb/.ssh/id_rsa.

Your public key has been saved in /home/robb/.ssh/id_rsa.pub.

The key fingerprint is:

SHA256:Eha16W8V+2m3Us/Q2ano4sOzZkYXmEfn6RRqp7qvwoI robb@robbLap

The key's randomart image is:

|      ...        |

|       . o . o   |

|      o o + = o  |

|     . o o = B   |

|      . S o O  .+|

|       . o + o.=o|

|    . . o = . *oo|

Aquí, opcionalmente, puede ingresar una frase de contraseña segura, que es altamente recomendable. Una frase de contraseña agrega una capa adicional de seguridad para evitar que usuarios no autorizados inicien sesión.

Ahora tiene una clave pública y privada que puede usar para autenticar. El siguiente paso es colocar la clave pública en su servidor para que pueda usar la autenticación basada en clave SSH para iniciar sesión.

Copiar las la llave publica al servidor

La forma más rápida de copiar su clave pública al host de Ubuntu es usar una utilidad llamada ssh-copy-id . Debido a su simplicidad, este método es altamente recomendado si está disponible. Si no tiene ssh-copy-id disponible para usted en su máquina cliente, puede usar uno de los dos métodos alternativos.

  • Copiar atraves de SSH.

  • Copiar manualmente la clave.

ssh-copy-id

Para copiarlas de esta manaera solo necesitamos introducir el siguiente comando 

ssh-copy-id username@remote_host(IP)

Respuesta:

/usr/bin/ssh-copy-id: INFO: 1 key(s) remain to be installed -- if you are prompted now it is to install the new keys

robb@192.168.0.151's password: 



Number of key(s) added: 1



Now try logging into the machine, with:   "ssh 'robb@192.168.0.151'"

and check to make sure that only the key(s) you wanted were added.

Una vez terminado esto ya podremos conectarnos utilizando el comando:

ssh user@server(ip)

Copia de archivos

Para el envio de archivos entre los servidores utilizaremos en terminal el comando scp el cual funciona de la siguiente manera:

scp direccionDeOrigen DireccionDeestion

Ejemplo del comando para obtener información desde nuestro servidor 

robb@tracer:~$ scp robb@192.168.0.151:~/prueba.txt prueba.txt

prueba.txt                                     100%   33    47.8KB/s   00:00

En este ejemplo estamos copiando a nuestra computador el archivo prueba.txt que se encuentra en el servidor.
